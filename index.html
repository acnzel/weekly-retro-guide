🧠 내 지식이 자라는 작업 세트

한 폴더 안에서 두 가지가 같이 자랍니다. 하나는 일하는 법(교훈 루프), 하나는 아는 것(지식 위키)이요. Claude Code로 일할수록 나도 AI도 똑똑해집니다. 개발 안 해본 분도 따라 할 수 있게 적었어요.

내 컴퓨터:
⏱️ 3분 요약 (끝까지 안 읽어도 흐름만):
① Obsidian으로 폴더 하나 만들기 → ② Claude Code(데스크탑 Code 탭) 설치 → ③ 설치 스크립트에서 폴더 경로 한 줄만 바꿔 실행 → ④ 평소처럼 일하면 교훈이 쌓이고, 금요일에 /weekly-retro로 검토 → ⑤ (선택) 자료를 raw/에 넣고 /ingest 하면 나만의 위키가 자랍니다.

이게 뭐예요?

AI와 일하다 보면 두 가지가 그냥 날아가 버려요. 내가 얻은 교훈, 그리고 내가 읽고 모은 지식이요. 이 세트는 그 둘을 같은 폴더 한 곳에 차곡차곡 쌓습니다. 쓰면 쓸수록 나도 AI도 똑똑해지고요.

두 앱이 같은 폴더를 공유합니다. Claude Code가 정리하고, 나는 Obsidian에서 봐요.

교훈 루프 Part A · 기본

일하며 얻은 교훈을 일지에 모읍니다. 그중 반복되는 것만 골라 AI의 영구 규칙으로 올려요. 같은 실수를 두 번 하지 않게요.

지식 위키 Part B · 확장

읽은 자료를 넣어 두면 AI가 주제별로 정리하고 이어 붙여, 나만의 백과사전으로 키웁니다.

왜 좋을까?
  • 나한테는 '결정·교훈·지식'이 검색 가능한 자산으로 쌓여요. "그때 왜 그렇게 정했더라?", "그 자료 어디 뒀더라?" 싶을 때 한 폴더에서 바로 찾습니다.
  • AI한테는 같은 실수를 반복하지 않고, 내가 모은 지식을 근거로 답하게 됩니다. 그래서 점점 똑똑해져요.
  • 개발자가 아니어도 됩니다. 문서·자료 정리, 분석, 기획, 투자, 취미 리서치까지 Claude Code로 하는 일이면 다 통해요.
한 줄로 비유하면 이렇습니다. 그냥 AI 채팅은 그때그때 찾아보고 버려서 아무것도 안 남아요. 이 세트는 넣을 때마다 쌓이고 서로 이어집니다. (Andrej Karpathy의 'LLM Wiki' 아이디어에 주간 회고를 더했습니다.)
📌 읽는 순서. 공통 준비를 먼저 하세요. Part A(교훈 루프)까지만 해도 충분히 쓸 만합니다. Part B(지식 위키)는 원할 때 같은 폴더에 이어서 더하면 돼요.

왜 이게 통하나 — AI 잘 쓰는 4가지

설치보다 이게 먼저예요. 도구가 아니라 원리를 알면, 이 세트를 넘어 어디서든 AI를 잘 부립니다.

  1. AI는 당신을 기억하지 못합니다. 새 대화는 늘 백지에서 시작해요. 그래서 읽힐 맥락(CLAUDE.md·위키·일지)을 내가 만들어 두고 다시 먹여야 합니다. 이 세트가 하는 일이 바로 그 '외장 기억'을 쌓는 거예요.
  2. 버리는 대화와 쌓이는 자산은 다릅니다. 그냥 채팅은 끝나면 사라지죠. 정리해서 파일로 남기면 다음 번이 그 위에서 시작합니다. 지식이 복리로 붙어요.
  3. 사람이 승인하고, AI가 노동합니다. AI가 멋대로 규칙을 만들거나 지우지 않아요. 후보를 보여주면 당신이 고릅니다. 통제가 있으니 믿고 맡길 수 있고요.
  4. 루프엔 리듬과 게이트가 필요합니다. 만들고(기록·수집) → 돌아보고(점검) → 추리고(승인) → 적용. 이 한 바퀴를 주기적으로 돌릴 때만 자랍니다.

하나만 기억한다면: 루프는 입력이 좋아야 복리가 됩니다. 대충 쌓으면 대충 자라요. 그래서 아래 각 단계마다 '좋은 입력' 요령을 같이 적어 뒀습니다.

한 달 뒤엔 이렇게 됩니다

처음 1~2주는 티가 잘 안 나요. 복리라서 그렇습니다. 쌓이면 이런 게 생겨요.

교훈 루프 → 승격된 규칙 (예시)

내 CLAUDE.md에 이런 한 줄이 늘어갑니다.
- 보고서 초안은 항상 '결론 먼저' 구조로 쓴다 #promoted
그다음부터 AI가 알아서 이 방식을 지켜요.

지식 위키 → 자란 페이지 (예시)

wiki/entities/거래처A.md 안에 미팅·결정·연락이 분기별로 쌓이고, /query "거래처A 그동안 흐름 정리" 한 줄에 타임라인으로 답이 나옵니다.

그래서 초반엔 결과보다 리듬이 중요해요. 눈에 안 보여도 2~3주는 그냥 굴려 보세요. /loop-status로 "잘 돌고 있나"만 가끔 확인하면 됩니다.

어떻게 작동하나요? (교훈 루프 기준)

✍️
1. 기록Claude Code가 일을 끝내면 '교훈'을 일지에 자동으로 적습니다 (작은 #태그와 함께).
🔍
2. 집계매주 금요일 14:30, 컴퓨터가 자동으로 2번 이상 반복된 교훈을 모아 '리뷰 대기' 목록을 만듭니다.
✅
3. 승인당신이 /weekly-retro 를 실행하면 후보를 보여주고, "이건 규칙으로" 라고 고르기만 하면 됩니다.
🧠
4. 적용승인된 규칙을 AI가 매 작업마다 기억해서, 같은 실수를 다시 하지 않습니다.
핵심 원칙. 한 번 한 실수는 규칙으로 안 만듭니다(괜한 잔소리 방지). 두 번째로 반복됐을 때 비로소 후보가 돼요. 단, 되돌리기 힘든 치명적 실수는 한 번이라도 올립니다. (위키의 /lint·/ingest도 똑같이 '사람이 승인하는' 방식으로 움직여요.)

어디서 써야 작동하나요? (꼭 확인하세요)

이 세트는 Claude Code에서 돌아갑니다. Claude Desktop 앱을 쓴다면 앱 안의 여러 탭 중에서 꼭 Code 탭을, 환경은 "내 컴퓨터(Local)"로 골라야 해요. (일반 대화 탭이 아닙니다.)

✅Claude Desktop 앱 → Code 탭 (Local)
CLI와 똑같은 ~/.claude 설정·스킬·훅을 공유하고 내 컴퓨터 파일에 접근해요. 터미널 없이 GUI로 쓰니 비개발자에게 추천합니다.
✅터미널 Claude Code (CLI)
개발자·고급 사용자용. 위와 설정을 공유하니 결과는 똑같아요.
❌Chat 탭(일반 대화)
Claude Code가 아닙니다. CLAUDE.md·스킬·훅·로컬 파일을 쓰지 않아 작동하지 않아요.
❌클라우드(Remote) 세션
Anthropic 서버 샌드박스에서 돌아 내 Obsidian 폴더를 못 봅니다. 꼭 Local을 고르세요.
⏰매주 자동 점검(스캔)
이건 늘 돕니다. 내 컴퓨터 스케줄러가 돌리니 Claude를 켜두지 않아도 돼요.
🗂️ 어디에 쌓이나 — 둘은 저장 방식이 다릅니다.
· 교훈·규칙은 전역이에요. 설정에 적힌 지식 베이스 폴더 한 곳에 모이고, 어느 프로젝트 창에서 일하든 그리로 갑니다.
· 위키 지식은 연 폴더 기준(지역)이에요. Claude Code로 지금 연 폴더의 wiki/에 쌓여서, 다른 폴더를 열면 위키도 따로 생깁니다.
→ 그래서 위 그림처럼 지식 베이스 폴더 하나를 열고 거기서 쓰면 둘이 같은 곳에 모여 제일 단순해요. 여러 프로젝트를 오갈 때 어떻게 되는지는 아래 FAQ에서 자세히 다룹니다.

공통 준비물

  • Obsidian 무료 메모 앱이에요. 교훈 일지와 위키가 여기 쌓입니다.
  • Claude Code 이 세트의 엔진입니다. Claude Desktop 앱의 Code 탭(비개발자 추천)이나 터미널 CLI, 둘 중 하나면 돼요. 설치하고 로그인까지 해 두세요.
  • Mac은 python3가 보통 깔려 있어요. 없으면 설치 안내가 뜹니다.
  • Windows는 따로 깔 게 없습니다. PowerShell이 기본으로 들어 있어요.

Obsidian 설치 + 지식 베이스 폴더 만들기

obsidian.md 에서 내려받아 설치하고, 보관함(Vault)을 하나 만듭니다. 그 안에 "지식 베이스" 폴더(예: MyBrain)를 하나 만들고, 그 폴더의 전체 경로를 복사해 두세요. 다음 단계에서 한 번 붙여넣습니다. (교훈 일지는 이 폴더 안 debriefs/ 에 저장돼요. Part B 위키를 깔면 같은 폴더에 raw/·wiki/가 더해져서 한 폴더로 모입니다.)

Mac 경로 예시: /Users/내이름/Documents/MyBrain

폴더에서 option 키를 누른 채 우클릭 → "경로 이름 복사"로 정확한 경로를 얻을 수 있어요.

Windows 경로 예시: C:\Users\내이름\Documents\MyBrain

탐색기에서 폴더를 Shift+우클릭 → "경로로 복사"로 정확한 경로를 얻을 수 있어요.

Claude Code 설치 + 로그인

방법 A. Claude Desktop 앱 비개발자 추천 (터미널 없이 GUI로 사용)

  • 데스크탑 앱 다운로드 → 설치 → 로그인 → 상단의 Code 탭 클릭.
  • Windows는 Code 탭을 처음 열기 전에 Git for Windows가 필요합니다(앱이 안내해 줍니다). 설치 후 앱을 재시작하세요.
  • 새 세션을 만들 때 환경 = "내 컴퓨터(Local)", 프로젝트 폴더 = Obsidian 보관함 폴더를 선택하세요. (Chat 탭·클라우드(Remote)는 작동 안 함 — 위 표 참고)

방법 B. 터미널 CLI (개발자·고급 사용자)

npm install -g @anthropic-ai/claude-code

설치 후 claude 를 실행해 로그인하세요.

두 방법은 같은 설정(~/.claude)을 공유해요. 어느 쪽으로 깔든 나머지 단계는 똑같이 적용됩니다.
※ 설치 방법은 버전에 따라 달라질 수 있으니 공식 문서를 기준으로 하세요.

설치 스크립트 실행 (자동 설치)

아래 스크립트가 교훈 루프 전체를 한 번에 설치합니다. 맨 위 한 줄(지식 베이스 폴더 경로)만 앞에서 복사한 전체 경로로 바꾼 뒤 실행하세요.

경로 넣는 법. Finder/탐색기에서 "경로 복사"로 얻은 전체 경로(/Users/이름/…C:\Users\이름\…)를 따옴표 안에 그대로 붙여넣으세요. ~ 나 %USERPROFILE% 로 시작하는 경로는 넣지 마세요. (잘못 넣거나 안 바꾸면 스크립트가 그 자리에서 알려주고 멈춥니다.)
실행 방법(둘 중 하나):
① 아래 다운로드 → 받은 .command 파일을 더블클릭 (차단되면 우클릭 → "열기").
② 또는 복사 → 터미널에 붙여넣고 Enter.
실행 방법: 아래 복사 → PowerShell을 열고 붙여넣은 뒤 Enter.
실행이 막히면 먼저 이 한 줄을 실행하세요: Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

💡 Claude Desktop 앱만 쓰는 분도 괜찮습니다. Code 탭의 통합 터미널(보기 메뉴 또는 Ctrl+`)을 열어 붙여넣으면 별도 터미널 앱 없이 설치돼요. Windows는 통합 터미널의 PowerShell에서 실행하세요.

폴더 딱 두 개만 기억하세요

  

    

      📥 raw/ 내가 넣는 곳
      
기사·PDF·이미지·메모 같은 원본을 그냥 던져 넣는 곳이에요. /ingest 하면 그 파일은 처리 완료 보관함(ingested/)으로 자동으로 옮겨갑니다. 그래서 raw엔 아직 처리 안 한 것만 남아요. (삭제가 아니라 이동이에요.)

    

    

      🧠 wiki/ AI가 채우는 곳
      
AI가 raw 자료를 읽어 정리하고 이어 붙여 만든 지식 페이지들이에요(인물·개념·요약·비교). 목차(index)와 기록(log)도 알아서 관리합니다.

    

  

  내지식베이스/   (교훈 루프와 같은 폴더!)
├── raw/        ← 처리할 자료를 넣는 곳 (아직 ingest 안 된 것만 남음)
├── ingested/   ← /ingest 끝난 원본이 자동 보관됨 (삭제 아님)
├── debriefs/   ← 작업 교훈 일지 (Part A 교훈 루프가 관리)
└── wiki/       ← AI가 정리해서 채웁니다
    ├── entities/      (인물·회사·제품·기술)
    ├── concepts/      (개념·주제)
    ├── sources/       (원본 요약)
    ├── comparisons/   (질문에 대한 분석 저장)
    ├── index.md       (목차 — 항상 먼저 봄)
    └── log.md         (변경 기록)

  
raw 폴더 사용법

  

    
    
      
  • 웹 기사는 Obsidian Web Clipper(브라우저 확장)로 한 번 클릭하면 raw로 저장돼요.
    • 
      
  • PDF·이미지·데이터는 그냥 raw/ 폴더에 끌어다 놓으면 됩니다.
    • 
      
  • 파일 이름은 2026-06-11-기사제목.md처럼 날짜로 시작하면 정리가 쉬워요(권장).
    • 
    

    raw에 넣기만 하면 끝이에요. 정리는 다음 /ingest 한 번으로 AI가 합니다.
    
💡 아무거나 넣지 마세요. 나중에 다시 찾고 싶은 것만 넣습니다. 잡다하게 쌓으면 위키도 잡다해져요. 한 달 뒤 "이건 왜 넣었지?" 싶을 자료는 그냥 패스.

  


  
명령은 딱 세 개

  

    
/ingest
자료 넣기
raw에 넣은 자료를 위키로 정리해요. 관련 페이지를 한꺼번에 갱신하고 기록을 남깁니다. 내용이 충돌하면 ⚠️로 표시하고요. 처리한 원본은 raw에서 ingested/로 옮겨 둬서, 뭐가 처리됐는지 한눈에 보입니다(삭제 아님).
예) raw/2026-06-11-기사.md 를 /ingest 해줘

    

    
/query
물어보기
위키를 근거로 답해요. 어느 페이지에서 나왔는지 알려주고, 쓸 만한 답은 위키에 저장할지 물어봅니다.
예) /query GPT-5가 시장에 줄 영향 정리해줘
💡 좋은 질문이 좋은 답을 부릅니다. "요약해줘"보다 "A와 B를 표로 비교해줘", "지난달 대비 뭐가 바뀌었어?"처럼 종합·비교를 시키면 위키의 진짜 값이 나와요. 답의 출처 페이지는 한 번 눈으로 확인하고요.

    

    
/lint
점검(주 1회)
모순·외톨이 페이지·빠진 연결·오래된 내용을 찾아 보여줘요. 그중 승인한 것만 고칩니다. "다음에 조사할 주제"도 제안하고요.
예) /lint, 교훈 루프 점검하는 날 같이 돌리면 좋아요.

  

  
셋 다 사람이 통제해요. AI가 멋대로 지우거나 바꾸지 않고, 정리한 내용을 보여준 다음에 진행합니다. (교훈 루프의 '승인 게이트'와 같은 방식이에요.)


  
'나만의 사서'에 비유하면

  

    
이 위키는 나만을 위해 일하는 도서관 사서를 한 명 두는 것과 같아요. 세 명령이 곧 사서가 하는 일입니다.

    
    
      
  • 📥 raw/ = 반납함. 읽을거리를 일단 쌓아두는 책 더미예요. 아직 정리 전이고요.
    • 
      
  • 🗂️ /ingest = 사서가 읽고 꽂기. 내용을 읽어 주제별 서가에 분류하고, 핵심을 요약 카드로 만들어 둬요. 책 한 권을 읽으면 관련된 여러 칸의 카드가 한꺼번에 갱신됩니다.
    • 
      
  • 🧠 wiki/ = 잘 정리된 서가. 주제별 카드가 서로 연결돼 꽂혀 있는 곳이에요. 넣을수록 두꺼워집니다.
    • 
      
  • ❓ /query = 사서에게 물어보기. "이 주제 정리해줘" 하면 서가를 뒤져 종합해 답해요. 인터넷 검색이 아니라 내가 모아둔 것이 기준입니다.
    • 
      
  • 🧹 /lint = 서가 주기 점검. 사서가 가끔 서가를 돌며 중복·모순·낡은 카드를 찾아 "이거 정리할까요?" 하고 물어봐요.
    • 
    

    
일반 AI 채팅에 파일을 매번 올리는 건 복사기 옆 아르바이트 같아요. 그때그때 찾아주지만 끝나면 다 잊죠. 위키는 전속 사서라 읽은 걸 계속 쌓아 둡니다. 그래서 쓸수록 똑똑해져요.

맨 위 한 줄(지식 베이스 폴더 경로)만 본인 전체 경로로 바꾼 뒤 실행하세요. (Part A를 이미 설치했으면 이 줄은 무시되고 같은 폴더를 자동으로 씁니다.) ~ 로 시작하면 안 되고, 안 바꾸면 스크립트가 알려주고 멈춥니다.

      
아래 다운로드 → .command 더블클릭(차단 시 우클릭→"열기"), 또는 복사 → 터미널에 붙여넣기.

      
아래 복사 → PowerShell에 붙여넣고 Enter. (막히면 Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned 먼저)

      
💡 데스크탑 앱만 쓰는 분은 Code 탭의 통합 터미널(Ctrl+`)에 붙여넣어도 됩니다.

⚠️ 실행 전, 맨 위 한 줄의 "여기에…붙여넣으세요"를 지식 베이스 폴더의 전체 경로로 바꾸세요. (교훈 루프를 이미 깔았다면 같은 폴더를 자동으로 씁니다.)

첫 자료, '나' 프로필부터 만들기

      
raw에 뭘 넣을지 막막하면 '나' 프로필을 첫 자료로 만드세요. 만드는 방법은 당신이 Claude를 어디서 써왔는지에 따라 갈립니다. 어느 쪽이든 결과를 복사해 raw/about-me.md 로 저장하면 됩니다.

      
🧭 나는 어느 쪽일까?

      · Claude Desktop 일반 대화(Chat)만 써왔다 / 잘 모르겠다 → B안(추천). Claude는 당신을 거의 기억하지 못하므로, 자료를 직접 줘서 정리시킵니다.

      · Code 탭(Local)이나 메모리를 켠 claude.ai를 오래 써왔다 → A안도 좋습니다. Claude가 이미 가진 기억을 끌어냅니다.

      

        

          
B안 · 자료를 주고 정리시키기누구나 · 추천

          

        

        
이력서·소개글, 최근 업무 문서·회의록, 인상 깊었던 Claude 대화 캡처 같은 본인 자료를 붙여넣어 정리시키는 방식이에요. 환경이나 기억과 상관없이 누구나 되고, 준 자료에만 근거하니 지어낼 위험이 없습니다.

        

        
      

      

        

          
A안 · Claude의 기억에서 끌어내기기억 기반

          

        

        
Code 탭(Local)에서 위키 폴더를 열고 쓰거나, 메모리를 켠 claude.ai에서 쓰세요. Code 탭이라면 CLAUDE.md·메모리 파일이 근거가 됩니다.

        
빈 새 채팅(메모리 OFF)에서는 하지 마세요. Claude가 모르는 걸 그럴듯하게 지어내, 사실이 아닌 내용이 위키에 박힙니다. 그런 경우엔 위 B안을 쓰세요.

        

        
      

      
답변에 (확인 필요)가 남아 있으면 그대로 저장하세요. 나중에 실제 자료로 채우면 됩니다. 이렇게 만든 about-me.md가 위키의 '나' 엔티티가 돼요.

    


    

      
첫 사용

      
    
        
  1. Claude Code(또는 데스크탑 앱 Code 탭 · Local)를 위키 폴더로 엽니다.
    2. 
        
  2. raw/ 에 자료를 하나 넣어요. 처음이라면 위에서 만든 about-me.md, 그다음부터는 기사·PDF 뭐든 좋습니다.
    3. 
        
  3. /ingest 실행 → AI가 wiki/에 정리합니다. 결과를 확인하세요.
    4. 
        
  4. 궁금한 걸 /query 로 물어보고, 일주일에 한 번 /lint 로 점검합니다.
    5. 
      

      
⚠️ 반드시 Code 탭 + 내 컴퓨터(Local)에서, 위키 폴더를 열고 사용하세요. (Chat 탭·클라우드(Remote)에서는 폴더 파일을 못 다룹니다.)

    

  


  


  
실제로 이렇게 씁니다

  
설치를 마쳤으면, 평소엔 이런 식으로 써먹습니다.

  

    💬 팀 Slack 대화를 지식으로 (자동 수집)
    
중요한 논의·결정이 Slack에 묻혀 사라지는 걸 막는 흐름. 채널 대화를 위키로 끌어와 쌓습니다. '나만의 사서'에게 매일 Slack을 읽혀 두는 것과 같아요.

    
먼저, Slack 커넥터를 한 번 연결 (앞의 'MCP 연결' 방식)

    
    
      
  • claude.ai나 Claude Desktop에서 설정 → 커넥터(Connectors) → Slack 검색 → 연결(Slack 로그인·권한 승인). 워크스페이스에 따라 Pro 이상이거나 관리자 승인이 필요할 수 있어요.
    • 
      
  • 연결되면 Code 탭(Local)에서도 Slack 도구가 잡힙니다. 안 보이면 Claude Code를 새 세션으로 다시 열거나, 설정에서 커넥터 연결을 한 번 더 확인하세요.
    • 
    

    
그다음, 모아서 위키로

    
    
      
  • 수집. Code 탭에서 이렇게 시켜요. "어제 #채널A·#채널B의 메시지와 스레드를 전부 가져와 raw/slack/2026-06-15-채널A.md처럼 채널별로 저장해줘" (날짜는 어제, 한국시간 기준)
    • 
      
  • 정리. /ingest 하면 wiki/entities/사람.md·wiki/concepts/논의주제.md 에 쌓이고, 핵심 결정은 기록으로 남아요.
    • 
      
  • 묻기. /query "지난주 우리 팀 주요 결정 5가지랑 그 맥락 정리해줘"
    • 
    

    
(선택) 한 줄 명령으로 만들기 /slack-digest

    
매번 프롬프트 치기 번거로우면, 아래 개별 파일에서 slack-digest-SKILL.md·channels.txt를 받아 두고 Claude에게 설치를 맡기세요. Code 탭(Local)에서 이렇게요.

    
"받아둔 slack-digest-SKILL.md와 channels.txt를 참고해서 ~/.claude/skills/slack-digest/에 slack-digest 스킬을 설치해줘 (스킬 파일은 SKILL.md로 저장)."

    
설치하고 나면 세 가지를 꼭 챙기세요.
      
· 새 세션으로 다시 열기. 스킬은 세션 시작 때 로드돼서, 설치 직후엔 안 보입니다. Claude Code를 새로 열어야 /slack-digest가 떠요.
      
· Slack 커넥터가 연결돼 있어야 동작합니다(맨 위에서 연결한 그거예요).
      
· channels.txt에 본인 채널 ID를 채워 두세요.
      
이러면 /slack-digest 한 줄로 어제치가 raw/slack/에 떨어집니다. (Gmail·캘린더 같은 다른 커넥터도 같은 식으로 나만의 명령을 만들 수 있어요.)

  

  

    🍷 취미·관심사 깊이 파기
    
와인·커피·러닝화·여행지… 블로그 글, 유튜브 자막, 후기를 raw에 모읍니다.

    
    
      
  • 넣기. "이 와인 리뷰 정리해줘" 하면 wiki/concepts/내추럴와인.md, wiki/entities/생산자이름.md 가 자동으로 갱신돼요.
    • 
      
  • 묻기. "지금까지 모은 것 중 2만 원대 데일리 와인을 비교표로 만들어줘"
    • 
    

  

  

    💼 흩어진 업무 자료 한곳에
    
회의록·이메일·기획안·참고 자료를 프로젝트별로 raw에 넣습니다.

    
    
      
  • 넣기. "이번 주 회의록 ingest" 하면 wiki/entities/거래처.md, wiki/concepts/이번분기목표.md 에 쌓여요.
    • 
      
  • 묻기. "A 프로젝트 지금까지의 결정사항과 남은 과제를 타임라인으로 정리해줘"
    • 
      
  • 점검. 주 1회 /lint로 "서로 어긋난 결정"이나 "빠진 후속조치"를 잡아냅니다.
    • 
    

  

  

    📈 투자 종목 추적
    
증권사 리포트 PDF, 공시, 뉴스, 실적 자료를 종목별로 raw에 모읍니다.

    
    
      
  • 넣기. "삼성전자 1분기 실적자료 ingest" 하면 wiki/entities/삼성전자.md 에 실적 추이·컨센서스 갭이 쌓여요.
    • 
      
  • 묻기. "내 5종목의 최근 3개월 주요 이벤트를 타임라인으로 만들어줘"
    • 
      
  • 점검. /lint가 "내가 아직 안 챙긴 새 이슈"를 짚어 줍니다.
    • 
    

  

  
공통점은 이거예요. 나는 자료를 모으고 좋은 질문만 하고, 분류·연결·요약은 AI가 합니다. 몇 달 모으면 "살아 있는 나만의 백과사전"이 돼요. (공부·자격증 노트, 육아·건강 정보까지 무엇이든 같은 방식입니다.)


  


  
주간 정비는 한 자리에서

  
둘 다 설치했다면, 금요일 알림이 오는 날 한 자리에서 정비하세요. 같은 '사람이 승인하는' 리듬입니다.

  

    
    
      
  • /weekly-retro 로 반복된 교훈을 영구 규칙으로 올려요(승인/기각).
    • 
      
  • /lint 로 위키의 모순·외톨이 페이지·낡은 내용을 점검합니다(승인한 것만 수정).
    • 
      
  • /loop-status 로 둘 다 잘 돌고 있는지 대시보드로 확인하고요.
    • 
    

    교훈(주관적·암묵지)과 위키(출처 있는 지식)는 한 폴더 안에서도 섞지 않고 따로 둡니다. 성격이 다른 지식이라서요.
  


  


  
개별 파일 (수동 설치용)

자주 묻는 질문 (FAQ)

  

    교훈 루프(Part A)랑 지식 위키(Part B), 뭐가 다른가요?
    
교훈 루프는 내가 일하며 얻은 교훈을 쌓아 같은 실수를 막아요. 지식 위키는 내가 읽고 모으는 외부 지식을 쌓고요. 같은 폴더에 있어도 섞지는 않습니다. 하나는 주관적 경험, 하나는 출처 있는 지식이라 성격이 다르거든요. 둘 다 "사람이 승인하고 AI가 정리한다"는 똑같은 방식으로 움직입니다.

  

  

    프로젝트마다 다른 폴더에서 쓰면 위키가 나뉘나요?
    
두 시스템이 저장 방식이 달라요. 이걸 모르면 헷갈리기 쉬우니 꼭 확인하세요.

    

      
위키 지식
/ingest·/query		지금 연 폴더 기준입니다. Claude Code로 연 폴더의 wiki/에 저장돼요. 그래서 프로젝트 폴더를 따로따로 열고 ingest하면 위키도 폴더마다 따로 쌓이고, 자동으로 합쳐지지 않습니다.

      
교훈·규칙
debrief·/weekly-retro		한 곳에 모입니다. 설정(~/.claude/weekly-retro.config)에 적힌 지식 베이스 폴더로 갑니다. 어느 프로젝트 창에서 일하든 교훈은 자동으로 그 한 곳에 쌓여요.

    

    
위키를 하나로 키우고 싶다면, 지식 베이스 폴더를 하나 정해 두고 /ingest·/query는 그 폴더를 연 창에서 하세요. 프로젝트 코딩은 각 폴더에서 따로 해도 됩니다(교훈은 어차피 중앙으로 모여요). 프로젝트에서 나온 자료를 중앙 위키에 넣고 싶으면, 그 자료를 중앙 폴더의 raw/(예: raw/projectA/)에 넣고 거기서 ingest하면 됩니다.

  

  

    꼭 둘 다 설치해야 하나요?
    
아니요. Part A(교훈 루프)만 써도 충분합니다. 읽은 자료까지 쌓고 싶을 때 Part B를 같은 폴더에 더하면 됩니다. 반대로 위키만 쓰고 싶어도 됩니다(설치 스크립트가 분리돼 있어요).

  

  

    꼭 개발 지식이어야 하나요?
    
아니요. 업무 자료, 리서치, 관심 주제(예: 마케팅·투자·요리 레시피)도 똑같이 됩니다. raw에 넣고 /ingest 하면 됩니다.

  

  

    실행이 막혀요 / 권한 오류가 나요
    

      
MacWindows
      .command 더블클릭 시 "확인되지 않은 개발자" 경고가 뜨면, 파일을 우클릭 → 열기를 선택하세요. 한 번만 하면 됩니다.
      PowerShell에서 스크립트 실행이 차단되면 먼저 Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned 를 실행하세요. 작업 스케줄러 등록이 실패하면 PowerShell을 "관리자 권한으로 실행"한 뒤 다시 시도하세요.

    

  

  

    Mac에서 "python3가 필요합니다"라고 나와요
    
터미널에서 xcode-select --install 를 실행해 설치한 뒤, 설치 스크립트를 다시 실행하세요.

  

  

    알림(팝업)이 안 떠요
    
알림은 보조 수단이에요. 안 떠도 괜찮습니다. 다음에 Claude Code를 열면 "리뷰 대기 후보가 있다"고 알려주거든요. Mac은 시스템 설정 → 알림에서 권한을 확인할 수 있어요.

  

  

    실행 요일/시간을 바꾸고 싶어요
    
설치 스크립트 맨 위의 시간 설정값을 바꾼 뒤 다시 실행하면 됩니다.
    RETRO_WEEKDAY(1=월…5=금…7=일), RETRO_HOUR, RETRO_MINUTE
    $RetroDay(예: 'Monday'), $RetroHour, $RetroMinute 값을 수정하세요.

  

  

    raw 자료가 사라지거나 바뀌면요?
    
규약(CLAUDE.md)에 "AI는 raw를 절대 수정하지 않는다"가 박혀 있습니다. raw는 원본 보관소로만 쓰고, /ingest 후엔 ingested/로 이동(삭제 아님)되며, 정리물은 전부 wiki/에만 쌓입니다.

  

  

    여러 사람이 쓰는데 안전한가요?
    
네. 규칙으로 '승격'하거나 위키를 고치는 건 항상 당신이 직접 승인해야만 일어납니다. 자동 점검은 후보를 '정리만' 할 뿐, 당신 허락 없이 바꾸지 않습니다.

  


  

    설치한 걸 제거하려면? (되돌리기)
    
자동화와 스킬만 지웁니다. 폴더 안 데이터(debriefs/·raw/·wiki/)는 그대로 남아요.

    
# 교훈 루프(Part A): 스케줄 + 스킬
launchctl bootout gui/$(id -u)/com.user.weekly-retro 2>/dev/null
rm -f ~/Library/LaunchAgents/com.user.weekly-retro.plist
rm -rf ~/.claude/skills/weekly-retro

# 위키(Part B)·공통 스킬
rm -rf ~/.claude/skills/ingest ~/.claude/skills/query ~/.claude/skills/lint \
       ~/.claude/skills/loop-status ~/.claude/skills/slack-digest

    
# 교훈 루프(Part A): 작업 + 스킬
Unregister-ScheduledTask -TaskName WeeklyRetro -Confirm:$false
Remove-Item "$env:USERPROFILE\.claude\skills\weekly-retro" -Recurse -Force

# 위키(Part B)·공통 스킬
"ingest","query","lint","loop-status","slack-digest" | ForEach-Object {
  Remove-Item "$env:USERPROFILE\.claude\skills\$_" -Recurse -Force -ErrorAction SilentlyContinue }

    
~/.claude/settings.json에 추가된 훅 한 줄과 전역 CLAUDE.md 아래쪽에 붙은 부분은 직접 지우면 됩니다. 데이터 폴더까지 없애려면 그 폴더를 직접 삭제하세요.
