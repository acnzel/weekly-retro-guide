🧠 주간 리트로 설치 가이드

AI(Claude Code)와 일하며 얻은 교훈을 자동으로 모아, 같은 실수를 두 번 반복하지 않게 만드는 시스템 — 비개발자용 따라하기.

내 컴퓨터:

이게 뭐예요?

똑똑한 비서가 당신의 업무 일지를 매주 금요일에 들춰보고, "이 실수 또 하셨네요 — 앞으로 규칙으로 정해둘까요?" 하고 물어보는 것. 당신이 "네"라고 한 것만 영구 규칙이 되어, AI가 다음부터 그 실수를 알아서 피합니다.

왜 좋을까?
  • 나에게 — 검색 가능한 '결정·교훈' 자산이 쌓입니다. "그때 왜 그렇게 정했더라?"를 일지에서 바로 찾습니다.
  • AI에게 — 같은 실수를 반복하지 않습니다. 매주 반복된 교훈만 골라 영구 규칙으로 올리니, AI가 점점 똑똑해집니다.
  • 꼭 개발이 아니어도 — 문서 작성, 자료 정리, 분석, 기획 등 Claude Code로 하는 모든 업무에 적용됩니다.

어떻게 작동하나요?

✍️
1. 기록Claude Code가 일을 끝내면 '교훈'을 일지에 자동으로 적습니다 (작은 #태그와 함께).
🔍
2. 집계매주 금요일 14:30, 컴퓨터가 자동으로 2번 이상 반복된 교훈을 모아 '리뷰 대기' 목록을 만듭니다.
✅
3. 승인당신이 /weekly-retro 를 실행하면 후보를 보여주고, "이건 규칙으로" 라고 고르기만 하면 됩니다.
🧠
4. 적용승인된 규칙을 AI가 매 작업마다 기억해서, 같은 실수를 다시 하지 않습니다.
핵심 원칙: 1번만 일어난 실수는 규칙이 되지 않습니다(잔소리 방지). 2번째 반복됐을 때만 후보가 됩니다. 단, 돌이키기 힘든 치명적 실수는 1번만으로도 올립니다.

어디서 써야 작동하나요? — 꼭 확인하세요

이 시스템은 Claude Code에서 돌아갑니다. Claude Desktop 앱을 쓰는 분이라면 앱 안의 세 탭(Chat · Cowork · Code) 중 반드시 Code 탭을, 그리고 "내 컴퓨터(Local)" 환경으로 사용해야 합니다.

✅Claude Desktop 앱 → Code 탭 (Local)
CLI와 똑같은 ~/.claude 설정·스킬·훅을 공유하고 내 컴퓨터 파일에 접근합니다. 비개발자에게 추천 — 터미널 없이 GUI로 사용.
✅터미널 Claude Code (CLI)
개발자·고급 사용자용. 위와 설정을 공유하므로 결과는 동일.
❌Chat 탭(일반 대화)
Claude Code가 아닙니다. CLAUDE.md·스킬·훅·로컬 파일을 쓰지 않아 작동하지 않습니다.
❌클라우드(Remote) 세션
Anthropic 서버에서 샌드박스로 돌아 내 Obsidian 폴더를 못 봅니다. 반드시 Local을 고르세요.
⏰매주 자동 점검(스캔)
항상 작동 — 내 컴퓨터의 스케줄러가 돌리므로 Claude를 켜두지 않아도 됩니다.

준비물

  • Obsidian — 무료 메모 앱. 교훈 일지가 여기 쌓입니다.
  • Claude Code — 이 시스템의 엔진. Claude Desktop 앱의 Code 탭(비개발자 추천) 또는 터미널 CLI 중 하나. 설치 + 로그인 필요.
  • Mac: python3 — 보통 내장. 없으면 설치 안내가 뜹니다.
  • Windows: 추가 설치 불필요 — PowerShell이 기본 내장입니다.

Obsidian 설치 + 일지 폴더 만들기

obsidian.md 에서 내려받아 설치하고, 보관함(Vault)을 하나 만듭니다. 그 안에 "Claude Logs" 같은 폴더를 하나 만들고, 그 폴더의 전체 경로를 복사해 두세요. 다음 단계에서 한 번 붙여넣습니다.

Mac 경로 예시: /Users/내이름/Documents/MyVault/Claude Logs

폴더에서 option 키를 누른 채 우클릭 → "경로 이름 복사"로 정확한 경로를 얻을 수 있어요.

Windows 경로 예시: C:\Users\내이름\Documents\MyVault\Claude Logs

탐색기에서 폴더를 Shift+우클릭 → "경로로 복사"로 정확한 경로를 얻을 수 있어요.

Claude Code 설치 + 로그인

방법 A — Claude Desktop 앱 비개발자 추천 (터미널 없이 GUI로 사용)

  • 데스크탑 앱 다운로드 → 설치 → 로그인 → 상단의 Code 탭 클릭.
  • Windows는 Code 탭을 처음 열기 전에 Git for Windows가 필요합니다(앱이 안내해 줍니다). 설치 후 앱을 재시작하세요.
  • 새 세션을 만들 때 환경 = "내 컴퓨터(Local)", 프로젝트 폴더 = Obsidian 보관함 폴더를 선택하세요. (Chat 탭·클라우드(Remote)는 작동 안 함 — 위 표 참고)

방법 B — 터미널 CLI (개발자·고급 사용자)

npm install -g @anthropic-ai/claude-code

설치 후 claude 를 실행해 로그인하세요.

방법 A와 B는 같은 설정(~/.claude)을 공유합니다. 어느 쪽으로 설치하든 이 가이드의 나머지 단계는 완전히 동일하게 적용됩니다.
※ 설치 방법은 버전에 따라 달라질 수 있으니 공식 문서를 기준으로 하세요.

설치 스크립트 실행 (자동 설치)

아래 스크립트가 모든 것을 한 번에 설치합니다. 맨 위 한 줄(교훈 폴더 경로)만 1단계에서 복사한 전체 경로로 바꾼 뒤 실행하세요.

경로 넣는 법: Finder/탐색기에서 "경로 복사"로 얻은 전체 경로(/Users/이름/…C:\Users\이름\…)를 따옴표 안에 그대로 붙여넣으세요. ~ 나 %USERPROFILE% 로 시작하는 경로는 넣지 마세요. (잘못 넣거나 안 바꾸면 스크립트가 그 자리에서 알려주고 멈춥니다 — 엉뚱한 폴더가 만들어지지 않아요.)
실행 방법(둘 중 하나):
① 아래 다운로드 → 받은 .command 파일을 더블클릭 (차단되면 우클릭 → "열기").
② 또는 복사 → 터미널에 붙여넣고 Enter.
실행 방법: 아래 복사 → PowerShell을 열고 붙여넣은 뒤 Enter.
실행이 막히면 먼저 이 한 줄을 실행하세요: Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

💡 Claude Desktop 앱만 쓰는 분도 괜찮아요 — Code 탭의 통합 터미널(보기 메뉴 또는 Ctrl+`)을 열어 여기에 붙여넣으면 별도 터미널 앱 없이 설치됩니다. Windows는 통합 터미널의 PowerShell에서 실행하세요.

설치 확인

      

        
터미널에서 아래를 실행해 스케줄이 등록됐는지 확인합니다(숫자/이름이 보이면 성공):

        
launchctl list | grep weekly-retro

      

      

        
PowerShell에서 아래를 실행해 작업이 등록됐는지 확인합니다(상태가 보이면 성공):

        
Get-ScheduledTask -TaskName WeeklyRetro

      

      
그리고 1단계에서 만든 교훈 폴더 안에 리뷰 대기 — 주간 리트로 (날짜).md 파일이 생겼다면 정상 작동입니다. 👍

문제 해결 (FAQ)

  

    실행이 막혀요 / 권한 오류가 나요
    

      
MacWindows
      .command 더블클릭 시 "확인되지 않은 개발자" 경고가 뜨면, 파일을 우클릭 → 열기를 선택하세요. 한 번만 하면 됩니다.
      PowerShell에서 스크립트 실행이 차단되면 먼저 Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned 를 실행하세요. 작업 스케줄러 등록이 실패하면 PowerShell을 "관리자 권한으로 실행"한 뒤 다시 시도하세요.

    

  

  

    Mac에서 "python3가 필요합니다"라고 나와요
    
터미널에서 xcode-select --install 를 실행해 설치한 뒤, 설치 스크립트를 다시 실행하세요.

  

  

    알림(팝업)이 안 떠요
    
알림은 보조 수단입니다. 안 떠도 괜찮습니다 — 다음에 Claude Code를 열면 "리뷰 대기 후보가 있다"고 자동으로 알려줍니다. Mac은 시스템 설정 → 알림에서 권한을 확인할 수 있어요.

  

  

    실행 요일/시간을 바꾸고 싶어요
    
설치 스크립트 맨 위의 시간 설정값을 바꾼 뒤 다시 실행하면 됩니다.
    RETRO_WEEKDAY(1=월…5=금…7=일), RETRO_HOUR, RETRO_MINUTE
    $RetroDay(예: 'Monday'), $RetroHour, $RetroMinute 값을 수정하세요.

  

  

    제거하려면?
    
launchctl bootout gui/$(id -u)/com.user.weekly-retro
rm -f ~/Library/LaunchAgents/com.user.weekly-retro.plist
rm -rf ~/.claude/skills/weekly-retro ~/.claude/scripts/weekly-retro-scan.py ~/.claude/hooks/weekly-retro-session-start.sh

    
Unregister-ScheduledTask -TaskName WeeklyRetro -Confirm:$false
Remove-Item "$env:USERPROFILE\.claude\skills\weekly-retro" -Recurse -Force
Remove-Item "$env:USERPROFILE\.claude\scripts\weekly-retro-scan.ps1","$env:USERPROFILE\.claude\hooks\weekly-retro-session-start.ps1" -Force

    
CLAUDE.md 와 settings.json 에 추가된 부분은 직접 지우면 됩니다(교훈 일지 파일은 그대로 보존됩니다).

  

  

    여러 사람이 쓰는데 안전한가요?
    
네. 규칙으로 '승격'하는 건 항상 당신이 직접 승인해야만 일어납니다. 자동 점검은 후보를 '정리만' 할 뿐, 당신 허락 없이 규칙을 바꾸지 않습니다.
