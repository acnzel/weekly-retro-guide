🧩 한 단계 더 — 내 지식이 쌓이는 위키 (LLM Wiki)

읽은 자료를 넣을수록 AI가 스스로 정리·연결해, 나만의 백과사전이 자라납니다. 비개발자용 따라하기.

내 컴퓨터:

왜 이게 다음 단계인가요?

앞 가이드(주간 리트로)는 "내가 일하며 얻은 교훈"을 쌓는 거였어요. 이번엔 "내가 읽고 모으는 자료·지식"을 쌓습니다. 둘은 짝입니다 — 하나는 일하는 법, 하나는 아는 것이 자라요.

한 줄 비유:
  • 그냥 검색(RAG) — 그때그때 찾아보고 버립니다. 아무것도 안 쌓여요.
  • LLM 위키 — 자료를 넣을 때마다 AI가 관련 페이지 여러 개를 동시에 정리·연결합니다. 지식이 영구히 쌓이고 서로 이어집니다.
핵심 역할 분담: 나는 좋은 자료를 모으고 좋은 질문만, 정리는 AI가. (Andrej Karpathy의 'LLM Wiki' 아이디어)

폴더 딱 두 개만 기억하세요

📥 raw/ — 내가 넣는 곳

기사·PDF·이미지·메모 등 원본을 그냥 던져 넣는 곳. /ingest 하면 그 파일은 처리 완료 보관함(ingested/)으로 자동 이동해서, raw엔 아직 처리 안 된 것만 남습니다. (삭제가 아니라 이동)

🧠 wiki/ — AI가 채우는 곳

AI가 raw 자료를 읽어 정리·연결해 만드는 지식 페이지들(인물·개념·요약·비교). 목차(index)와 기록(log)도 자동 관리.

내지식베이스/ (주간 리트로와 같은 폴더!) ├── raw/ ← 처리할 자료를 넣는 곳 (아직 ingest 안 된 것만 남음) ├── ingested/ ← /ingest 끝난 원본이 자동 보관됨 (삭제 아님) ├── debriefs/ ← 작업 교훈 일지 (주간 리트로가 관리) └── wiki/ ← AI가 정리해서 채웁니다 ├── entities/ (인물·회사·제품·기술) ├── concepts/ (개념·주제) ├── sources/ (원본 요약) ├── comparisons/ (질문에 대한 분석 저장) ├── index.md (목차 — 항상 먼저 봄) └── log.md (변경 기록)

raw 폴더 사용법

  • 웹 기사 — Obsidian Web Clipper(브라우저 확장)로 한 번 클릭하면 raw로 저장됩니다.
  • PDF·이미지·데이터 — 그냥 raw/ 폴더에 끌어다 넣으세요.
  • 이름 규칙(권장) — 2026-06-11-기사제목.md 처럼 날짜로 시작하면 정리가 쉬워요.
raw에 넣기만 하면 끝. 정리는 다음 /ingest 한 번으로 AI가 합니다.

명령은 딱 세 개

/ingest
자료 넣기
raw에 넣은 자료를 위키로 정리. 관련 페이지들을 동시에 갱신하고, 충돌하면 ⚠️ 표시, 기록을 남깁니다. 처리한 원본은 raw에서 ingested/로 옮겨져 무엇이 처리됐는지 한눈에 보입니다(삭제 아님).
예) raw/2026-06-11-기사.md 를 /ingest 해줘
/query
물어보기
위키를 근거로 답합니다. 어느 페이지에서 나왔는지 알려주고, 좋은 답은 위키에 영구 저장할지 물어봅니다.
예) /query GPT-5가 시장에 줄 영향 정리해줘
/lint
점검(주 1회)
모순·고아 페이지·빠진 연결·오래된 내용을 찾아 보여주고, 승인한 것만 고칩니다. "다음에 조사할 주제"도 제안.
예) /lint — 주간 리트로 하는 날 같이 돌리면 좋아요.
셋 다 사람이 통제합니다. AI가 멋대로 지우거나 바꾸지 않고, 정리한 내용을 보여준 뒤 진행해요. (주간 리트로의 '승인 게이트'와 같은 철학)

따라하기 — 설치

준비 (이미 돼 있을 거예요)

Claude Code(데스크탑 앱 Code 탭 추천)와 Obsidian. 앞 가이드를 따라 설치했다면 그대로 쓰면 됩니다. 주간 리트로와 같은 '지식 베이스' 폴더를 씁니다 — 위키의 raw/·wiki/가 그 폴더 안에 함께 들어갑니다.

💡 주간 리트로를 이미 설치했다면, 아래 설치 스크립트가 같은 폴더를 자동으로 이어받습니다 — 경로를 다시 넣을 필요 없이 그냥 실행만 하면 됩니다. (처음 설치하는 경우에만 폴더 경로를 넣습니다.)
예) /Users/이름/Documents/MyBrainFinder에서 폴더 option+우클릭 → "경로 이름 복사"
예) C:\Users\이름\Documents\MyBrain탐색기에서 폴더 Shift+우클릭 → "경로로 복사"

설치 스크립트 실행

맨 위 한 줄(지식 베이스 폴더 경로)만 본인 전체 경로로 바꾼 뒤 실행하세요. (주간 리트로를 이미 설치했으면 이 줄은 무시되고 같은 폴더를 자동으로 씁니다.) ~ 로 시작하면 안 되고, 안 바꾸면 스크립트가 알려주고 멈춥니다.

아래 다운로드 → .command 더블클릭(차단 시 우클릭→"열기"), 또는 복사 → 터미널에 붙여넣기.
아래 복사 → PowerShell에 붙여넣고 Enter. (막히면 Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned 먼저)

💡 데스크탑 앱만 쓰는 분은 Code 탭의 통합 터미널(Ctrl+`)에 붙여넣어도 됩니다.

install-llm-wiki-mac.commandMac · 원클릭

첫 자료 — '나'에 대한 프로필부터 만들기

      
raw에 뭘 넣을지 막막하면 '나' 프로필을 첫 자료로 만드세요. 만드는 방법은 당신이 Claude를 어디서 써왔는지에 따라 갈립니다. 어느 쪽이든 결과를 복사해 raw/about-me.md 로 저장하면 됩니다.


      
🧭 나는 어느 쪽일까?

      · Claude Desktop 일반 대화(Chat)만 써왔다 / 잘 모르겠다 → B안(추천). Claude는 당신을 거의 기억하지 못하므로, 자료를 직접 줘서 정리시킵니다.

      · Code 탭(Local)이나 메모리를 켠 claude.ai를 오래 써왔다 → A안도 좋습니다. Claude가 이미 가진 기억을 끌어냅니다.


      

        

          
B안 · 자료를 주고 정리시키기누구나 · 추천

          

        

        
이력서·소개글, 최근 업무 문서·회의록, 인상 깊었던 Claude 대화 캡처 등 본인 자료를 붙여넣고 정리시키는 방식. 환경·기억과 무관하게 누구나 되고, 준 자료에만 근거하므로 지어낼 위험이 없습니다.

        

        
      


      

        

          
A안 · Claude의 기억에서 끌어내기기억 기반

          

        

        
Code 탭(Local)에서 위키 폴더를 열고 쓰거나, 메모리를 켠 claude.ai에서 쓰세요. Code 탭이라면 CLAUDE.md·메모리 파일이 근거가 됩니다.

        
빈 새 채팅(메모리 OFF)에서는 하지 마세요. Claude가 모르는 걸 그럴듯하게 지어내, 사실이 아닌 내용이 위키에 박힙니다. 그런 경우엔 위 B안을 쓰세요.

자주 묻는 질문

  

    주간 리트로(교훈 일지)랑 뭐가 다른가요?
    
주간 리트로는 내 작업 습관/교훈을 쌓아 같은 실수를 막는 거고, LLM 위키는 내가 읽고 모으는 외부 지식을 쌓는 겁니다. 폴더도 따로 두는 걸 추천해요. 둘 다 "사람이 승인, AI가 정리"라는 같은 철학으로 움직입니다.

  

  

    꼭 개발 지식이어야 하나요?
    
아니요. 업무 자료, 리서치, 관심 주제(예: 마케팅·투자·요리 레시피)도 똑같이 됩니다. raw에 넣고 /ingest 하면 됩니다.

  

  

    /lint 를 자동으로 돌릴 수 있나요?
    
지금은 직접 /lint 를 입력하는 방식입니다(주 1회 권장). 앞 가이드의 '금요일 자동 점검' 알림을 받는 날 같이 돌리면 습관 들이기 좋아요.

  

  

    raw 자료가 사라지거나 바뀌면요?
    
규약(CLAUDE.md)에 "AI는 raw를 절대 수정하지 않는다"가 박혀 있습니다. raw는 원본 보관소로만 쓰고, 정리물은 전부 wiki/에만 쌓입니다.
